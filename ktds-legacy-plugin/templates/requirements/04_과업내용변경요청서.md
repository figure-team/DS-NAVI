# 과업내용변경요청서 (Change Request, CR)

> **용도**: 확정된 요구사항/과업의 **변경·철회**를 공식 요청하고 승인 근거를 남긴다(절차 B).
> **작성 단위**: **변경요청 1건당 이 파일 1개** (CR 번호 단위)
> **주 사용자**: 발주처 · PM · PL
>
> ## 📌 ID 체계
> - **변경요청ID `CR-{일련번호}`** = 변경/철회 요청 1건 (예: CR-001 "카카오 로그인 철회")
> - 대상은 **요청ID(REQ)** 단위로 기재한다. REQ 철회 시 하위 요구사항(SFR/SIR/DAR/SER…)이 **동반 폐기**된다.
>
> ## 📌 생성 규칙 (AI 작성 지침)
> - `{ }` 안의 안내 문구는 실제 내용으로 **모두 치환**한다. 빈 칸 금지.
> - **요구사항/문서를 삭제하지 않는다.** 철회는 상태를 `폐기(CR-xxx)`로 바꿔 이력을 보존한다.
> - 대상 요청ID·요구사항ID는 [요구사항 목록표]와 **철자까지 동일**하게 유지한다.
> - 영향 범위·후속조치는 [변경영향분석서]에서 상세화한다(여기서는 요약).
