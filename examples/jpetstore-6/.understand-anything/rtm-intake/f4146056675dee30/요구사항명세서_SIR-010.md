## 2. 요구사항 상세 명세

| 항목 | 내용 |
|---|---|
| **요구사항ID** | SIR-010 |
| **소속 요청ID** | REQ-001 |
| **요구사항명** | 네이버 OAuth2 연동 |
| **상태** | 유효 |
| **구분 / 우선순위** | 인터페이스 / 상(必) |
| **정의** | 네이버 OAuth2 인증서버와 연계하여 인가요청·인가코드-액세스토큰 교환·프로필 조회를 수행하는 인터페이스 요구사항. |
| **상세 기능** | ① 네이버 OAuth2 인증서버로 인가요청(authorize)을 전송한다.<br>② 콜백에서 인가코드(code)를 수신한다.<br>③ 인가코드를 액세스 토큰으로 교환한다.<br>④ 액세스 토큰으로 네이버 프로필 API(닉네임·이메일)를 조회해 SFR-010 에 반환한다. |
| **입력 데이터** | 네이버 애플리케이션 인증정보(client_id/secret/redirect_uri, 설정값), 콜백 인가코드 |
| **출력 데이터** | 액세스 토큰(내부 처리용, 세션 비저장 — SER-010), 프로필 데이터(이메일·닉네임) |
| **처리 흐름** | 인가요청 전송 → 사용자 네이버 인증·동의 → 콜백(code) 수신 → 토큰 교환 API 호출 → 프로필 조회 API 호출 → 결과(이메일·닉네임)를 SFR-010 계정 연동 단계로 전달 |
| **선행 요구사항** | - |
| **예외 / 제약** | 네이버 인증서버 응답 지연·오류 / 액세스 토큰 만료 / 이메일 제공 동의 거부(스코프 미동의) |
| **인수 기준(완료조건)** | ① 인가코드→액세스토큰 교환이 정상 응답 시 성공률 100%로 완료된다. ② 프로필 조회 응답에서 이메일·닉네임을 정상 추출한다. |
| **검증 방법** | 시험 — 네이버 개발자센터 테스트 앱으로 인가요청/콜백/토큰교환/프로필조회 전 구간 연동 시험 |
| **출처 / 추적** | SFR-010(네이버 소셜 로그인)에서 파생 / to-be:auth/naver-oauth-client(신규) / 시험케이스 미정([추정]) |
