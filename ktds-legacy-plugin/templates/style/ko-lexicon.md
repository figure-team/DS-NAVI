# 표기 통일 렉시콘 (ko-lexicon)

`fill-merge` 가 산문 필드(요약·진술·설명·라벨)에 **결정론 문자열 치환**으로 적용한다.
문맥을 보지 않으므로 **문맥 없이도 항상 옳은 표기만** 담는다 — 문맥 판단이 필요한
번역투 재작성은 문체 검수 라운드(LLM)의 몫이다. 인용(citations/evidence/snippet)은
절대 치환되지 않는다.

- 프로젝트 override: `.understand-anything/templates/style/ko-lexicon.md`(있으면 이 파일 대신 **단독** 적용).
- 통일 표기 칸에 `(삭제)` 를 쓰면 해당 표기를 제거한다.
- 편집 후 재빌드 불필요 — 다음 `fill-merge` 부터 반영된다.

| 금지 표기 | 통일 표기 | 비고 |
|---|---|---|
| 되어진다 | 된다 | 이중 피동 |
| 되어지는 | 되는 | 이중 피동 |
| 되어질 | 될 | 이중 피동 |
| 되어졌다 | 됐다 | 이중 피동 |
| 되어집니다 | 됩니다 | 이중 피동 |
| 데이타 | 데이터 | 표기 통일 |
| 어플리케이션 | 애플리케이션 | 표기 통일 |
| 밸리데이션 | 검증 | 음차 |
