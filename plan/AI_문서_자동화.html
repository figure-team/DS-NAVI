Overview

개요

THE PITCH

"기존 프로젝트에 명령 한 번이면, 코드 근거가 붙은 운영 문서가 자동으로 만들어집니다."

Understand-Anything(오픈소스) 기반 Claude Code 플러그인. 별도 프로그램 설치 없이, 쓰던 AI CLI 명령으로 바로 사용합니다.

📌 이번 MVP는 "근거 기반 문서 자동화 + 검토/승인"을 증명하는 단계입니다.

📄

Core Value

핵심 가치 2가지

VALUE 01

근거 기반 분석

모든 문장에 근거: 파일:라인이 붙고, 근거가 없으면 저장 자체가 차단됩니다.

LoginController는 /login 처리 [확정(AI)]
근거: LoginController.java:42
VALUE 02

검토·승인·감사

AI 초안(DRAFT)을 담당자가 검토·승인(APPROVED)하고, 전 과정이 감사 로그로 남습니다.

DRAFT → UNDER_REVIEW → APPROVED
audit log · 누가·언제·무엇을
How it works

시스템 흐름

프로젝트 코드
AI 분석 — 지식 그래프 → 근거 매핑 → 문서화코드 근거가 붙은 문장만 통과
근거 붙은 문서 5종  (DRAFT)
담당자 검토·승인  →  APPROVED ✓

각 문장엔 근거가, 각 단계엔 감사 로그가 자동으로 따라붙습니다.

Use case 01

최초 분석

상황 — 프로젝트 PM이 (인수받은/유지보수 중인) 프로젝트를 처음으로 분석해 기준 문서 세트를 만든다.

terminal
$ /understand-init     # 초기화 (VCS 자동 감지, 설정 생성)
$ /understand          # 코드베이스 스캔 → 지식 그래프 생성
$ /understand-docs     # 근거 기반 5종 문서 자동 생성
01_tech-stack
언어·프레임워크·라이브러리
02_architecture
레이어·모듈 의존 관계
03_feature-spec
업무 기능·진입점
04_api-spec
API·처리 Controller
05_db-spec
테이블·컬럼·Mapper
→ docs/ 즉시 생성
근거 링크로 구조 파악
Use case 02

문서 신뢰성 검토 · 승인

상황 — AI가 쓴 문서를 공식 운영 문서로 채택하기 전 검증이 필요하다.

terminal
$ /understand-docs review --list                     # 검토 대기 + [추정] 수
$ /understand-docs review --doc 04_api-spec.md        # [추정] → [확정(홍길동)]
$ /understand-docs approve --doc 04_api-spec.md --by "홍길동"   # 공식 승인
4단계 신뢰도 태그
[확정(AI)] [확정(담당자)] [추정] [확인 필요]
판단 어려우면 위임
동적 SQL·리플렉션은 [확인 필요]로 담당자에게
전 과정 감사 기록
생성·검토·승인이 audit log에 남음
Use case 03  ·  MVP+

변경 영향도 분석 MVP+

$ /understand-impact --path src/auth/LoginService.java   # 파일 지정
