# SI 배치정의서

> 상태: DRAFT · ktds doc-generator · 근거 기반 자동 생성

## 배치 목록

batch-jobs.json(W2 결정론 스캔) 1건 = 표 1행 — **정적 스캔 인벤토리 기반 초안**입니다.
트리거·스케줄·핸들러·도달범위는 코드/설정 근거(file:line) → [확정].
배치명(초안)은 추론 → [추정], 사람이 업무명으로 확정.
**데이터대상·선행/후행·수행서버·재기동/실패처리는 정적 분석이 알 수 없는 운영 지식**이라
전부 [미확인] — 운영 이관/감리 제출 전에 반드시 사람이 채웁니다.
'해석'은 잡 구현 파일의 정적 해석 여부만 뜻함(배치 검증/운영 여부 아님):
해석됨 | [미확인] | 외부(shell·crontab — 프로젝트 밖 실행체).
도달범위(파일)는 정적 파일 의존 BFS 근사(도달성 리포트와 동일 depth 상한) — 미해석 행은 [미확인].
BAT_ID 는 내용 파생 안정 id(재스캔에도 동일 배치 = 동일 id).
