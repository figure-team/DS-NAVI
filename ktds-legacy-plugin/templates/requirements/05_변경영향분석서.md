# 변경영향분석서 (Change Impact Analysis)

> **용도**: 변경요청(CR)의 영향을 RTM 역추적으로 분석해 설계/코드/데이터/시험 파급과 후속조치를 식별한다(절차 B).
> **작성 단위**: **변경요청 1건당 이 파일 1개** (CR 번호 단위)
> **주 사용자**: PL · 개발자 · 테스터 · 감리
>
> ## 📌 생성 규칙 (AI 작성 지침)
> - 데이터 소스: `rtm-intake.mjs impact <projectRoot> <REQ>` 의 구조화 결과(영향 기능 분류·의존·AC·산출물·후속조치).
> - `{ }` 안의 안내 문구는 실제 내용으로 **모두 치환**한다. 빈 칸 금지. **추적 칸은 비우지 않는다.**
> - 영향 기능 분류: **회귀**(구현 코드 존재 — 원복 시 회귀시험) / **계획취소**(미착수 TO-BE) / **타요구유지**(다른 유효 요구가 사용).
> - 후속조치는 체크리스트로 두고, 완료 시 별도 기록한다(여기서는 식별만).
