# /understand-report

> 🌐 **언어:** 사용자에게 보여주는 모든 설명은 **한국어**로 한다.

기간 실적 요약의 단일 명령. 결정론 수집 스크립트를 실행하고 결과를 보고한다:

```
node ${CLAUDE_PLUGIN_ROOT}/scripts/understand-report.mjs <projectRoot> [--weeks N | --month YYYY-MM | --range A..B]
```

- 기간 인자가 없으면 **최근 1주**(`--weeks 1`).
- 사용자가 "6월 실적", "지난달"처럼 말하면 `--month YYYY-MM` 으로, "이 태그부터"면 `--range A..B` 로 번역해 실행한다.

## 산출물
- `.spec/map/work-summary.json` — 기계 산출(커밋/모듈/진척 집계, 재현 앵커 포함).
- `.understand-anything/doc-output/si-실적요약보고서.md` + `.xlsx` — 대시보드 산출물 탭에서 열람·편집·확정.

## 핵심 원칙 (절대 준수)
- **날조 0**: 실적 하이라이트는 수집 수치를 고정 문형에 끼운 결정론 문장이다. 너는 커밋에 없는 내용을 요약에 추가하지 않는다.
- 수집이 [미확인]을 남기면(git 불가·원장 없음·커밋 범위 모드의 진척) 그대로 보고한다 — 0 과 [미확인]은 다르다.
- **보강 서술(경영진 관점 하이라이트 등)을 요청받으면**: 생성된 문서를 읽고 제안문을 `[추정]` 마킹으로 보여주되, 반영은 **대시보드의 문서 편집·확정 플로우로만** 안내한다(엔진 산출물·doc-output 파일을 직접 수정하지 않는다).
- 코드를 수정하지 않는다.

## 완료 보고
스크립트 출력(기간·커밋/파일 수·RTM/문서 진척·근거율)을 요약해 전하고, 문서 위치와
대시보드 산출물 탭 확인 방법을 안내한 뒤 끝낸다. 날짜 축은 committer date(랜딩 시점),
주간 윈도는 HEAD 커밋 시각 앵커의 반개구간임을 한 줄로 덧붙인다(재현 가능성 설명).
