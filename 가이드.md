# DS-NAVI 사용 가이드

레거시 시스템을 분석해 대시보드로 이해·추적·보고하는 전 과정을 기능별로 설명한다.
"왜 이 기능이 있는가"는 [README.md](README.md)를, 설계 상세는 `docs/ktds/*.md`를 참조.

> **호스트별 설치·실행 절차는 별도 가이드로:** 🤖 [Claude Code](가이드-claude.md) · ⚡ [opencode](가이드-opencode.md)
> — 커맨드 이름은 양쪽이 같지만 설치·헤드리스 실행·대시보드 연동 방법이 다르다. 이 문서는 호스트 무관 공통(대시보드·메뉴) 사용법이다.

---

## 1. 시작하기

### 1-1. 분석 실행 (분석 대상 프로젝트에서)

Claude Code에 플러그인을 설치한 뒤, 분석할 프로젝트 루트에서 순서대로 실행한다.

| 순서 | 스킬 | 산출물 | 비고 |
|---|---|---|---|
| 0 | `/understand-init` | `understanding.config.json`, `.spec/` scaffold | 프로젝트 초기화(설정·마스터 인덱스) |
| 1 | `/understand-map` | `.spec/map/` (domain-map, db-schema, crud-matrix …) | 업무 지도·구조·데이터 맵의 원천 |
| 2 | `/understand-screens` | `.spec/map/screens.json` + 캡처 | 화면설계서(앱 기동 가능해야 캡처) |
| 3 | `/understand-rtm` | `.spec/map/rtm.json` | 요구사항 추적표 원장 |
| 4 | `/understand-impact` | `.spec/map/impact.json` | 변경·영향 분석(CR 단위) |
| 5 | `/understand-policy` | `policy-signals.json`, `policy-reconcile.json` | 정책서 신호·대조 |
| 6 | `/understand-docs` | `.spec/wiki/` SI 산출물 문서 | 산출물 탭 원천 |
| 7 | `/understand-report` | `work-summary.json` | 주간 실적 보고서 |

- 2~7은 필요한 것만 골라 실행해도 된다. 각 메뉴는 산출물이 없으면 "산출물 없음"을 정직하게 표시한다(0건과 미실행을 구분).
- `/understand`(U-A 지식그래프 생성)는 ktds 파이프라인에서 **은퇴**했다(구조 메뉴는 map 산출로 렌더 — `docs/ktds/STRUCTURE_FROM_MAP_DESIGN.md`). 지식그래프 메뉴의 자유 탐색·투어가 필요할 때만 선택 실행.

### 1-2. 대시보드 실행

```bash
# 라이브(분석 대상 프로젝트를 직접 서빙 — 편집·확정·인테이크 가능)
UNDERSTAND_ACCESS_TOKEN=<토큰> GRAPH_DIR=<분석한 프로젝트 경로> pnpm dev:dashboard

# 데모(정적 스냅샷 — 읽기 전용)
pnpm --filter @understand-anything/dashboard build:demo && pnpm --filter @understand-anything/dashboard preview:demo
```

- 기동 로그에 `Dashboard URL: http://127.0.0.1:<포트>/?token=…`이 출력된다. **모든 데이터 엔드포인트는 토큰 게이트**라 URL의 `?token=`이 없으면 403.
- 토큰은 첫 진입 시 sessionStorage에 저장되고 URL에서 제거된다. 이후 딥링크는 토큰 없이 동작.
- 편집·확정 기능은 라이브 + 토큰 보유 시에만 노출된다(데모는 읽기 전용).

### 1-3. QA 파라미터

| 파라미터 | 효과 |
|---|---|
| `?onboard=skip\|force` | 온보딩 건너뛰기/강제 |
| `?theme=<presetId>` | 1회성 테마 프리셋(저장 안 됨) |

## 2. 공통 조작 (모든 메뉴 공통)

**URL이 곧 상태다.** 탭·선택·검색·필터가 전부 URL 쿼리로 관리된다 → 새로고침·뒤로가기에도 유지되고, URL을 복사해 공유하면 같은 화면이 열린다.

**통합 검색.** 각 메뉴 상단의 검색창은 이름·경로·내용을 함께 매칭하고 일치 부분을 하이라이트한다. 검색 중에는 "상위 N건" 절단이 해제되어 전체가 노출된다.

**근거 칩 → 코드 뷰어.** `파일명:라인` 형태의 근거 칩을 클릭하면 코드 뷰어가 해당 위치로 열린다. 모든 수치·판정의 출처를 즉시 확인하는 것이 이 도구의 핵심 사용법이다.

**신뢰도 라벨 읽는 법.**
- `근거확보` — 기계(정적 분석)가 코드 근거를 확보한 판정. **사람 확정이 아니다**(툴팁에 "기계 판정" 명시).
- `추정` — 기계가 간접 신호로 추론한 값. 검수 대상.
- `확정` — **사람이 검수·확정 버튼으로 승인한 것**(확정자·시각 기록). 기계 판정과 시각적으로 구분된다.

**편집·확정.** 라이브 모드에서 제목·설명 등을 인라인 편집하고 확정할 수 있다. 편집 내용은 원본 산출물이 아닌 오버레이 파일(`*-overrides.json`)에 저장되므로 **재분석해도 사람의 확정이 살아남는다**.

**테마.** 좌측 하단 테마 버튼. 기본은 DS-NAVI 라이트(붉은 액센트), 다크 프리셋 선택 가능.

## 3. 메뉴별 사용법

### 홈 `/`
프로젝트 전체 요약 — 이번 주 실적 카드, 위험 상위 카드, 프로그램 타일. 각 카드는 해당 상세 메뉴로 라우팅되는 진입점이다.

### 업무 지도 `/domains/:domainId?flow=`
코드에서 복원한 **업무 도메인 지도**. 도메인 카드 → 시스템 구성도(노드·업무 칩) → 업무 흐름도(프로세스별 순서도, `businessFlows` 복수 장) 순으로 내려간다.
- 도메인 클릭 → 상세(근거·기능 목록). 흐름도는 `?flow=`로 장 전환.
- 다른 메뉴(변경영향·프로그램)의 도메인 칩에서 이 페이지로 딥링크된다.

### 구조 `/structure?node=&level=&overlay=`
지식그래프 기반 **시스템 구조 지도**(React Flow). 레이어·도메인 클러스터로 계층 탐색.
- `?node=`로 특정 노드 포커스(화면설계서 "구조 탭에서 보기" 등에서 진입).
- `?overlay=diff|impact|risk` — 변경 diff / 변경영향 / 위험 등급을 노드 위에 오버레이. 변경영향·품질위험 메뉴의 "그래프 오버레이" 버튼이 이 딥링크를 쓴다.
- 노드 클릭 → 우측 패널(정보·파일)에서 상세와 소스 열람.

### 데이터 `/data?tab=&table=&q=&crudq=&crudTable=&pivot=`
DB 스키마·CRUD 매트릭스·코드 테이블 3탭.
- **테이블 탭**: 테이블·컬럼(코멘트 포함) 통합 검색, 상세에서 컬럼 정의·행 데이터 샘플·코드성 판정 사유.
- **CRUD 탭**: 기능×테이블 매트릭스. 기능 검색·테이블 필터·전치(행/열 뒤집기) 토글. DB 접근 미검출 행은 접이식으로 분리. 셀 클릭 → 근거 팝오버 → 코드 뷰어.
- **코드 탭**: 코드성 테이블 값 목록과 판정 사유.
- 상단 미해석 배너: 경고(warn)와 참고(info)를 분리해 사유별로 묶어 보여준다.

### 화면설계서 `/screens?screen=&q=`
실행 중인 앱을 실측 캡처한 화면 명세. 좌측 도메인 트리(검색 가능) → 우측 캡처+번호 배지+범례 표.
- 배지(입력/버튼·이벤트/링크)와 범례 표 행은 서로 hover 연동된다.
- 범례의 근거 칩 클릭 → 핸들러 코드로 점프. `graphNodeId`가 있으면 "구조 탭에서 보기".
- 헤더에 도달 실패·미매핑·프래그먼트 건수가 표시된다(숨기지 않음).
- 라이브에서 제목·설명 인라인 편집, 화면 단위 확정.

### 추적표 `/rtm?view=&fn=&ts=&req=&q=`
요구사항 추적표(RTM). 4개 뷰 — **기능 기준 / 요청 기준 / 시험 시나리오 / 커버리지 현황**.
- 기능 행 클릭 → 드로어에서 진입점·구현·데이터(CRUD)·테스트 추적 셀을 근거와 함께 열람, 셀 단위 편집·확정.
- **새 요청(인테이크)**: `+ 새 요청` → 6단계 가이드(①식별 → ②영향분석 → ③목록표 → ④정의서 → ⑤명세서 → ⑥RTM 편입). 각 단계는 컨펌해야 다음으로 진행되고, 진행 중 세션은 `?sid=`로 복원된다. ①의 `[확인필요]` 질문에 답하면 식별이 답을 반영해 개정된다.
- **변경 관리**: 확정된 요구의 변경 요청·철회(이력 보존, 삭제 아님).
- 시나리오 확정은 당시 G/W/T 전문을 스냅샷으로 박제한다 — 재생성돼도 확정본이 조용히 바뀌지 않는다.

### 변경·영향 `/change?q=&via=`
최신 변경영향 분석(CR 단위) 1건을 요약. 변경 기점(seeds) → 상류/하류 파일 → **영향받는 진입점(API)·도메인·플로우·건드리는 DB(매퍼/테이블)**.
- 파일 검색·경유(viaKind) 필터. "확인 필요" 항목은 사유별 접이식.
- 도메인/플로우 칩 → 업무 지도로, "그래프 오버레이" → 구조 탭 impact 오버레이로 점프.
- 새 분석은 추적표 인테이크 또는 우상단 "자연어 영향 분석"으로 시작한다.

### 프로그램 `/programs?ptab=&pq=&ptype=&pdomain=`
프로그램 목록(인벤토리) 4탭 — 목록 / FP 산정 근거 / 인터페이스 / 배치.
- 목록: 유형 StatTile 클릭=필터, 검색(이름·경로·ID·도메인·비고), 도메인 신뢰도 배지(도달성=근거확보, 디렉토리 추론=추정), 도메인 셀 → 업무 지도 점프.
- FP 탭: 요약 + **후보 상세 표 2개**(트랜잭션 기능 후보 — 왜 미분류인지, 데이터 기능 ILF — 어느 테이블인지, 근거 포함). ILF 이름 → 데이터 탭 점프.
- 인터페이스/배치 탭: 프로토콜·트리거 분포와 미해석 건수 표면화. 0건은 "스캔 완료·0건"으로 미실행과 구분.

### 품질·위험 `/quality?tab=&q=`
3탭 — 위험 모듈 / 분석 커버리지 / 정확도 기준선.
- 위험 모듈: 복잡도·팬인·변경량 합성 점수 Top N. 검색·등급/레이어/유형 필터 시 전체 노출. 행별 [확정]/[추정] 배지(미측정 사유 툴팁), 열 헤더 클릭 정렬, 행 → 코드 뷰어. "그래프 오버레이" → 구조 탭 risk 오버레이.
- 커버리지: 언어×기능 지원 매트릭스, 레이어 분포, 부분 지원 파일 수, 도달성(도달/미도달), FP 집계 — **분석이 어디까지 믿을 만한지**를 보여준다.
- 정확도 기준선: 골든셋 동결 시 산출물별 정밀도. 미동결이면 안내 카드(정상).

### 보고서 `/report?sort=&author=&q=&open=`
주간 실적 보고서(git + 원장 수집 사실만, 날조 0).
- 하이라이트 문장 + 직전 기간 대비 증감(커밋·라인·파일 `a→b(±d)`), 모듈별 변경(출처 배지), 주간 커밋 추이, **커밋 이력 표**(정렬·작성자/메시지 필터·행 펼침 → 파일 목록 → 코드 뷰어).
- 우상단: **요약 복사**(클립보드) / **인쇄**(보고서만 인쇄되는 print 스타일) / md 다운로드.
- git 이력이 얕거나(shallow) 없으면 상단에 수치 불완전 경고가 뜬다.

### 정책서 `/policy?tab=&q=&status=`
코드·DB에서 추출한 정책 신호와 기존 정책서 대조. 3탭 — 카테고리별 / 도메인 / 대조 현황.
- 카테고리별: 데이터 신호(테이블명 그룹핑·종류 소계)·용어사전, 근거 칩 → 코드 뷰어. 신호 0인 카테고리는 "미해석 사유"(신호 미발견 vs 추출 미구현)로 구분 표기.
- 도메인: 도메인 정책서 문서 카드 → 산출물 탭으로.
- 대조: 일치/불일치/누락/미정의 StatTile 클릭=필터, 전건 스크롤 열람. 기존 정책서 미입수 시 배너 안내.

### 산출물 `/deliverables/:docId?q=&status=`
템플릿 기반 SI 산출물 문서(마크다운) 열람·편집·확정.
- 좌측 트리: 방법론 폴더 + 검색 + 상태 칩(전체/확정/초안).
- 본문: 근거 칩(`파일:라인` → 코드 뷰어)·[추정] 배지 시각화, 긴 문서는 목차(TOC) 앵커.
- 편집: 미저장 변경은 "미저장" 표시 + 이탈 시 확인. 저장=확정(확정자 인라인 입력, 기억됨). 편집은 오버레이 저장이라 재생성에 생존. xlsx 병기 문서는 원본 변경 시 stale 경고.

### 문서(위키) `/wiki` · 지식그래프 `/knowledge`
- 위키: `.spec/wiki/` 문서 리더(산출물과 달리 참조용 열람 중심).
- 지식그래프: 원본 Understand Anything의 그래프 워크벤치(노드 정보·파일 탐색기·투어).

## 4. 문제 해결

| 증상 | 원인/조치 |
|---|---|
| 403 Forbidden | URL에 `?token=` 누락. 기동 로그의 Dashboard URL로 재진입 |
| 메뉴에 "산출물 없음" | 해당 스킬 미실행 — §1-1 표에서 스킬 실행 |
| 편집/확정/새 요청 버튼이 없음 | 데모 모드이거나 토큰 없음. 라이브 dev로 실행 |
| 코드 뷰어 "소스 불가" | 그래프 밖 파일(allowlist) — 분석 범위에 포함시켜 재분석 |
| RTM 인테이크가 진행 안 됨 | 라이브 dev 전용(정적 preview는 job 엔드포인트 404) |
| 화면설계서 캡처 깨짐 | 캡처 자산 경로/토큰 문제 — 폴백 카드에 경로 표시됨 |
