## 2. 요구사항 상세 명세

| 항목 | 내용 |
|---|---|
| **요구사항ID** | SIR-050 |
| **소속 요청ID** | REQ-003 (환불 추가) |
| **요구사항명** | 환불 요청 화면 UI |
| **상태** | 유효 [추정] |
| **구분 / 우선순위** | 인터페이스(SIR) / 중 |
| **정의** | 기존 주문 조회 화면(viewOrder)에 환불 요청 버튼을 추가해 사용자가 환불 절차를 시작할 수 있는 UI 진입점을 제공한다. |
| **상세 기능** | ① 환불 요청 버튼 노출 — 기존 주문 조회 화면(viewOrder)에 환불 요청 버튼을 추가한다 (to-be:screen/order-refund-button)<br>② 환불 요청 진입 동선 — 버튼 클릭 시 SFR-030 환불 요청 흐름으로 진입한다<br>③ 환불 불가 주문 처리 — 환불 불가 조건(3일 초과·환불 진행/완료)에서의 버튼 노출·비활성 정책은 상세 설계에서 확정한다 |
| **입력 데이터** | 주문 조회 화면 표시 데이터(주문번호·주문일·주문 상태) |
| **출력 데이터** | 환불 요청 버튼 UI 노출, 클릭 시 환불 요청 화면/흐름 전환 |
| **처리 흐름** | 주문 조회(flow:ANY /actions/Order.action?viewOrder) → 환불 가능 여부 판단(주문일·상태) → 환불 요청 버튼 노출 → 클릭 → SFR-030 환불 요청 흐름 진입 |
| **선행 요구사항** | SFR-030 (주문 환불 기능) |
| **예외 / 제약** | · 별도 환불 관리 화면은 신설하지 않는다(Q-3 답변 '버튼 추가')<br>· 관리자용 환불 현황 화면은 범위 외<br>· 화면 축 근거가 인테이크 번들에 포함되지 않아 버튼 배치 위치는 [추정]이다 — 상세 설계에서 화면설계서와 정합 확인 필요 |
| **인수 기준(완료조건)** | · 본인 주문의 주문 조회 화면에서 환불 요청 버튼 노출률 100%<br>· 버튼 클릭 시 환불 요청 흐름 진입 성공률 100% |
| **검증 방법** | 시연 + 시험 (TC-SIR-050-01~02 [추정] — 버튼 노출 / 환불 흐름 진입) |
| **출처 / 추적** | 선행 요구사항 SFR-030 파생 + [확인필요] 답변 Q-3(기존 화면에 버튼 추가) / 대상 화면: `/actions/Order.action?viewOrder` (rtm.json flow 실재 대조 통과) / TC-SIR-050-01~02 [추정] |
