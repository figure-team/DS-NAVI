## 2. 요구사항 상세 명세

| 항목 | 내용 |
|---|---|
| **요구사항ID** | SER-060 |
| **소속 요청ID** | REQ-003 (환불 추가) |
| **요구사항명** | 환불 권한 제어 |
| **상태** | 유효 [추정] |
| **구분 / 우선순위** | 보안(SER) / 상(必) |
| **정의** | 로그인 사용자가 본인 주문에 대해서만 환불을 요청할 수 있도록 환불 요청 시 주문 소유자 권한을 검증한다. |
| **상세 기능** | ① 주문 소유자 검증 — 환불 요청 시 로그인 세션 사용자와 주문 소유자의 일치를 검증한다 (to-be:order/authorize-refund)<br>② 불일치 거절 — 타인 주문에 대한 환불 요청은 거절하고 오류를 응답한다<br>③ 기존 정책 확장 — 본인 주문 조회 제한 정책(PL-007, `OrderActionBean.java:178`)의 검증 방식을 환불 동작에 동일하게 적용한다 |
| **입력 데이터** | 로그인 세션 사용자 계정, 환불 대상 주문의 소유자 정보 |
| **출력 데이터** | 권한 검증 결과(허용/거절), 거절 시 오류 응답 |
| **처리 흐름** | 환불 요청 수신 → 로그인 세션 사용자 확인 → 주문 소유자 대조(기존 viewOrder 의 본인 주문 검증과 동일 방식) → 일치 시 SFR-030 환불 흐름 진행, 불일치 시 거절 |
| **선행 요구사항** | SFR-030 (주문 환불 기능) |
| **예외 / 제약** | · 미로그인 사용자는 기존 인증 정책에 따라 로그인 화면으로 유도한다<br>· 관리자 권한 체계 신설·역할 기반 접근 제어(RBAC) 전면 도입은 범위 외 |
| **인수 기준(완료조건)** | · 타인 주문에 대한 환불 요청 거절률 100%(우회 성공 0건)<br>· 본인 주문에 대한 정상 환불 요청 허용률 100% |
| **검증 방법** | 시험 (TC-SER-060-01~02 [추정] — 타인 주문 거절 / 본인 주문 허용) |
| **출처 / 추적** | 선행 요구사항 SFR-030 파생 + 기존 정책 policy-domain-order.md §4 PL-007(본인 주문 조회 제한) / 근거 코드: `src/main/java/org/mybatis/jpetstore/web/actions/OrderActionBean.java:178` / TC-SER-060-01~02 [추정] |
