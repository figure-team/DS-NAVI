## ▸ Slide 1 — 무엇인가

> # Understand Anything
>
> ### "코드를 읽지 않고 이해한다 — 코드베이스를 인터랙티브 지식 그래프로"

**한 줄 정의**
정적분석(tree-sitter) **+** LLM을 결합해, 코드베이스를 **자동으로 분석·시각화**하는 지식 그래프로 변환

**구현 현황**

- 🌐 10여 개 언어 지원 (UA 분석 엔진 + 대시보드)
- ✅ 실제 레거시(**JPetStore**, Spring/MyBatis)에서 **4대 기능 검증**

> 🎙️ _발표자 노트:_ "개인이 며칠 걸려 파악하던 걸, 도구가 자동으로 그려주고 팀이 공유합니다. 뒤에 보여드릴 4개 기능은 전부 실제로 돌려서 데이터가 나온 것들입니다."

## ▸ Slide 2 — 구현한 4대 기능

> ### 하나의 분석으로, 이해의 전 과정을 커버

| 기능                    | 무엇을 보여주나                                                                          |
| ----------------------- | ---------------------------------------------------------------------------------------- |
| **① 구조 그래프**       | 전체 코드 → 아키텍처 그래프 (파일·클래스·함수·의존성, 레이어 자동 분류 + 소스 뷰어·검색) |
| **② 도메인 지도** ★ktds | 업무 도메인 → 흐름 → 실행 경로 (API→Service→DAO→DB), **메서드 호출 순서**까지            |
| **③ 변경 영향도** ★ktds | 바뀐 코드가 **무엇에 영향 주는지** (정/역방향 도달성)                                    |
| **④ 문서/위키** ★ktds   | 명세서(기능, API, DB 등) **자동 생성** (전부 `파일:라인` 출처 인용)                      |

> 🎙️ _발표자 노트:_ "②③④가 KTDS가 직접 설계·구축한 레거시 특화 영역입니다."

## ▸ Slide 3 — 어떻게 동작하는가

> ### 결정적 정적분석 + LLM, 그리고 검증 — 4단 파이프라인

```
  코드베이스
     │
  ┌──▼─────────────┐   ┌────────────────┐   ┌──────────────┐   ┌─────────────────┐
  │ ① 정적분석       │──▶│ ② LLM 의미 보강  │──▶│ ③ 기계 검증    │──▶│ ④ 지식 그래프     │
  │ tree-sitter     │   │ 요약·도메인·이름  │   │ 인용 ↔ 소스    │   │ → 대시보드 시각화 │
  │ 구조 100% 확정   │   │ (파일:라인 인용)  │   │ 환각 자동 차단  │   │ (JSON, 팀 공유)  │
  └─────────────────┘   └────────────────┘   └──────────────┘   └─────────────────┘
   파일·클래스·함수·       "무엇을 하는지"를       소스와 대조해           4대 산출물:
   호출 관계 추출          LLM이 채움             틀린 건 [확인 필요]      구조·도메인·영향도·문서
```

**핵심 설계**

- **결정적 우선** — 구조(노드·호출)는 정적분석이 확정, LLM은 _의미만_ 채움 → 재현 가능
- **인용 강제 + 검증** — 모든 설명에 `파일:라인` 근거, 기계가 소스와 대조해 **환각 차단**
- **메서드 단위 호출 그래프** — 실제 호출을 따라가 흐름 정확도 확보

**시연**

- ✅ 4대 기능 + 대시보드 동작, JPetStore 실데이터 검증

> 🎙️ _발표자 노트:_ "정리하면 — 구조는 도구가 정확히 뽑고, 의미는 LLM이 채우되 전부 출처로 검증합니다. 이 파이프라인으로 4대 기능을 실제 레거시에서 동작까지 확인했고, 이제 더 큰 사내 코드로 확장합니다."
