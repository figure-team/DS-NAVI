## 2. 개요

- **목적**: JPetStore 6 온라인 반려동물 쇼핑몰에 고객 요청(REQ-003 "환불 추가")에 따른 주문 환불 기능을 신설한다. 현 시스템에는 주문 취소/환불 처리 로직이 존재하지 않아(policy-domain-order.md §8 미결 사항) 전부 신규 개발이다.
- **범위**: 포함 — 환불 요청 접수(전체/부분), 환불 자동 처리, 환불 상태값 3종 신설·자동 전이, 주문 조회 화면 환불 버튼, 본인 주문 한정 권한 제어. 제외 — 결제수단 실승인취소(PG 연동 부재), 관리자 수동 승인, 별도 환불 관리 화면, RBAC 전면 도입.
- **용어 정의**: 환불 상태 3종 = REFUND_REQUESTED(환불 요청) → REFUND_PROCESSING(처리 중) → REFUNDED(완료). viewOrder = 주문 조회 화면(`/actions/Order.action?viewOrder`). PL-007 = 본인 주문 조회 제한 정책(policy-domain-order.md §4).

## 4. 요청별 요구사항 정의

### REQ-003 환불 추가
- **요청 개요**: 고객이 주문에 대한 환불 기능 추가를 요청했다. [확인필요] 답변으로 부분 환불 지원(Q-1), 자동 승인(Q-4), 주문 후 3일 이내 제한(Q-5), 기존 화면에 버튼 추가(Q-3), 환불 상태 3단계 전이(Q-2·Q-6)가 확정됐다.
- **출처/요청일**: 고객 요청 / 요청일 미기재 (인테이크 세션 da4bdcd7f059a53a, 2026-07-16)

#### [SFR-030] 주문 환불 기능 — 우선순위: 상 · 상태: 유효 [추정]
- **정의**: 생성된 주문에 대해 사용자가 전체 또는 라인아이템 단위 부분 환불을 요청하고, 요청된 환불이 관리자 개입 없이 자동으로 처리되는 기능을 제공한다.
- **범위**: 포함 — 환불 요청 접수(전체/부분 환불), 환불 자동 처리, 주문 후 3일 이내 요청 가능 시간 제한 검증. 제외 — 실제 결제수단 승인취소(PG 연동 — 현 시스템에 결제 게이트웨이 연동 없음), 관리자 수동 승인 프로세스(Q-4 답변 '자동'으로 제외). ※ 현 코드에는 주문 취소/환불 로직이 부재해(policy-domain-order.md §8 미결 사항) 전부 신규 [추정]이다.
- **출처/관련**: 고객 요청 REQ-003 '환불 추가' + [확인필요] 답변 Q-1(부분 환불 지원)·Q-4(자동 승인)·Q-5(3일 이내) / 관련 코드: `OrderService.java`, `OrderActionBean.java`

#### [DAR-040] 주문 상태 확장 (환불 상태) — 우선순위: 상 · 상태: 유효 [추정]
- **정의**: 주문 상태 체계에 환불 관련 상태값 3종(REFUND_REQUESTED 환불 요청, REFUND_PROCESSING 처리 중, REFUNDED 완료)을 추가하고 순차 자동 전이를 정의한다.
- **범위**: 포함 — ORDERSTATUS 상태값 3종 추가, REFUND_REQUESTED → REFUND_PROCESSING → REFUNDED 자동 전이(Q-6 답변 '전자'). 제외 — 기존 주문 데이터 마이그레이션(신규 상태는 신규 환불 건에만 적용), 상태값 코드 그룹/enum 체계 전면 재설계(현행은 미정의 상태 — policy-domain-order.md §3).
- **출처/관련**: 선행 요구사항 SFR-030 파생 + [확인필요] 답변 Q-2('알아서 해줘' — 설계 위임)·Q-6('전자' — 3단계 전이 확정) / policy-domain-order.md §3

#### [SIR-050] 환불 요청 화면 UI — 우선순위: 중 · 상태: 유효 [추정]
- **정의**: 기존 주문 조회 화면(viewOrder)에 환불 요청 버튼을 추가해 사용자가 환불 절차를 시작할 수 있는 UI 진입점을 제공한다.
- **범위**: 포함 — 주문 조회 화면(`/actions/Order.action?viewOrder`)에 환불 요청 버튼 노출 및 환불 요청 진입 동선. 제외 — 별도 환불 관리 화면 신설(Q-3 답변 '버튼 추가'로 제외), 관리자용 환불 현황 화면.
- **출처/관련**: 선행 요구사항 SFR-030 파생 + [확인필요] 답변 Q-3(기존 화면에 버튼 추가)

#### [SER-060] 환불 권한 제어 — 우선순위: 상 · 상태: 유효 [추정]
- **정의**: 로그인 사용자가 본인 주문에 대해서만 환불을 요청할 수 있도록 환불 요청 시 주문 소유자 권한을 검증한다.
- **범위**: 포함 — 환불 요청 시 주문 소유자 검증(기존 본인 주문 조회 제한 정책 PL-007 을 환불 동작으로 확장 적용, `OrderActionBean.java:178` 근거). 제외 — 관리자 권한 체계 신설, 역할 기반 접근 제어(RBAC) 전면 도입.
- **출처/관련**: 선행 요구사항 SFR-030 파생 + 기존 정책 policy-domain-order.md §4 PL-007(본인 주문 조회 제한)
