## 2. 요구사항 상세 명세

| 항목 | 내용 |
|---|---|
| **요구사항ID** | SFR-030 |
| **소속 요청ID** | REQ-003 (환불 추가) |
| **요구사항명** | 주문 환불 기능 |
| **상태** | 유효 [추정] |
| **구분 / 우선순위** | 기능(SFR) / 상(必) |
| **정의** | 생성된 주문에 대해 사용자가 전체 또는 라인아이템 단위 부분 환불을 요청하고, 요청된 환불이 관리자 개입 없이 자동으로 처리되는 기능을 제공한다. |
| **상세 기능** | ① 환불 요청 접수 — 생성된 주문에 대해 주문 전체 또는 라인아이템 단위 부분 환불 요청을 접수한다 (to-be:order/refund-request)<br>② 환불 가능 기간 검증 — 주문일로부터 3일 이내인지 검증하고, 초과 시 요청을 거절한다 (to-be:order/validate-refund-window)<br>③ 환불 자동 처리 — 접수된 환불 요청을 관리자 개입 없이 자동으로 처리하고 결과를 기록한다 (to-be:order/process-refund) |
| **입력 데이터** | 주문번호(orderId), 환불 범위(전체 / 부분 — 부분이면 라인아이템 번호·수량), 로그인 세션 사용자 |
| **출력 데이터** | 환불 요청 접수 결과(성공/거절 사유), 주문 상태 갱신(DAR-040 환불 상태 3종), 환불 처리 내역 |
| **처리 흐름** | 주문 조회(viewOrder) → 환불 요청 버튼 진입(SIR-050) → 주문 소유자 권한 검증(SER-060) → 환불 가능 기간 검증(주문 후 3일 이내) → 환불 접수(REFUND_REQUESTED) → 자동 처리(REFUND_PROCESSING → REFUNDED, DAR-040) |
| **선행 요구사항** | - (본 요청의 최상위 요구사항 — DAR-040·SIR-050·SER-060 이 본 요구사항에서 파생) |
| **예외 / 제약** | · 주문 후 3일 초과 건은 환불 요청을 거절한다(Q-5 답변)<br>· 타인 주문에 대한 환불 요청은 거절한다(SER-060 위임)<br>· 이미 환불 요청·처리 중·완료된 주문의 중복 환불 요청은 거절한다<br>· 실제 결제수단 승인취소는 범위 외 — 현 시스템에 PG 연동이 없어 내부 상태 처리까지만 수행한다<br>· 현 코드에 주문 취소/환불 로직이 부재(policy-domain-order.md §8 미결)하여 본 명세는 전부 신규 [추정]이다 |
| **인수 기준(완료조건)** | · 전체 환불·부분 환불 요청 시험 시 접수·자동 처리 성공률 100%<br>· 주문 후 3일 초과 주문에 대한 환불 요청 거절률 100%<br>· 환불 요청 접수부터 처리 완료까지 관리자 수동 개입 0회<br>· 환불 요청 화면 응답시간 3초 이내 [추정] |
| **검증 방법** | 시험 (TC-SFR-030-01~04 [추정] — 전체 환불 / 부분 환불 / 3일 초과 거절 / 자동 처리 완료) |
| **출처 / 추적** | 고객 요청 REQ-003 '환불 추가' + [확인필요] 답변 Q-1(부분 환불)·Q-4(자동 승인)·Q-5(3일 이내) / 관련 코드: `src/main/java/org/mybatis/jpetstore/service/OrderService.java`, `src/main/java/org/mybatis/jpetstore/web/actions/OrderActionBean.java` / 정책: policy-domain-order.md §8(환불 로직 미결) / 영향분석: 세션 impact-run 1efc46db06501569 / TC-SFR-030-01~04 [추정] |
